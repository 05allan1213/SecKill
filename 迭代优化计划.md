## 一、架构层次简化 ✅

### 1.1 当前架构问题

**当前架构**：
```
Server -> Logic -> Service -> Biz -> Data
（5层架构）
```

**问题分析**：
- 层次过多，职责边界模糊
- Logic层和Service层功能重复
- Biz层和Model层概念混淆
- 增加了代码复杂度和维护成本

**目标架构**：
```
Server -> Logic -> Model
（3层架构，符合go-zero标准）
```

### 1.2 架构重构方案

#### Step 1: 移除Service层

**当前代码结构**：
```
internal/
├── server/          # gRPC服务器
├── logic/           # 业务逻辑层（go-zero标准）
├── service/         # 业务服务层（Kratos遗留）
├── biz/             # 业务领域层（Kratos遗留）
└── data/            # 数据访问层
```

**目标代码结构**：
```
internal/
├── server/          # gRPC服务器
├── logic/           # 业务逻辑层（合并service和biz）
└── model/           # 数据模型层（重命名data）
```

### 2.5 日志级别规范

**日志级别使用规范**：

| 级别 | 使用场景 | 示例 |
|------|----------|------|
| **Debug** | 开发调试信息 | "processing request", "cache hit" |
| **Info** | 正常业务流程 | "user login success", "order created" |
| **Warn** | 警告信息，不影响主流程 | "slow query detected", "retry attempt" |
| **Error** | 错误信息，影响业务流程 | "database connection failed", "payment failed" |
| **Fatal** | 致命错误，程序需退出 | "config load failed", "port bind failed" |

**示例**：
```go
// Debug - 开发调试

// Info - 正常业务

    logx.Field("orderNum", orderNum),
    logx.Field("userID", userID),
)

// Warn - 警告
logx.Warn("slow database query",
    logx.Field("sql", sql),
    logx.Field("duration", duration),
)

// Error - 错误
logx.Error("failed to create order",
    logx.Field("userID", userID),
    logx.Field("goodsID", goodsID),
    logx.Field("err", err),
)

// Fatal - 致命错误
logx.Fatal("failed to connect database", logx.Field("err", err))
```

### 2.6 日志中间件优化

**优化拦截器日志**（seckill-main/cmd/sec_kill/interceptors.go）：
```go
func NewAccessLogInterceptor() grpc.UnaryServerInterceptor {
    return func(ctx context.Context, req interface{}, info *grpc.UnaryServerInfo, handler grpc.UnaryHandler) (interface{}, error) {
        begin := time.Now()
        reply, err := handler(ctx, req)
        
        // 使用结构化日志
        logger := logx.WithContext(ctx)
        fields := []logx.LogField{
            logx.Field("method", info.FullMethod),
            logx.Field("duration", time.Since(begin).Milliseconds()),
            logx.Field("req", req),
        }
        
        if err != nil {
            fields = append(fields, logx.Field("err", err))
            logger.Errorw("rpc call failed", fields...)
        } else {
            logger.Infow("rpc call completed", fields...)
        }
        
        return reply, err
    }
}
```

package config

import (
    "context"
    "encoding/json"
    "time"
    
    "github.com/zeromicro/go-zero/core/stores/etcd"
)

type ConfigManager struct {
    client *etcd.EtcdClient
}

func NewConfigManager(etcdConf etcd.EtcdConf) (*ConfigManager, error) {
    client, err := etcd.NewEtcdClient(etcdConf)
    if err != nil {
        return nil, err
    }
    return &ConfigManager{client: client}, nil
}

func (m *ConfigManager) WatchConfig(ctx context.Context, key string, onChange func(value string)) {
    go m.client.Watch(key, func(key, value string) {
        onChange(value)
    })
}

func (m *ConfigManager) GetConfig(ctx context.Context, key string) (string, error) {
    value, err := m.client.Get(key)
    if err != nil {
        return "", err
    }
    return value, nil
}

package middleware

var (

        Help:    "Duration of seckill requests",
        Buckets: prometheus.DefBuckets,

func PrometheusMiddleware() grpc.UnaryServerInterceptor {
    return func(ctx context.Context, req interface{}, info *grpc.UnaryServerInfo, handler grpc.UnaryHandler) (interface{}, error) {
        start := time.Now()
        resp, err := handler(ctx, req)
        
        duration := time.Since(start).Seconds()
        status := "success"
        if err != nil {
            status = "error"
        }
        
        seckillCounter.WithLabelValues(info.FullMethod, status).Inc()
        seckillDuration.WithLabelValues(info.FullMethod).Observe(duration)
        
        return resp, err
    }
}

func openDB(conf cfg.DatabaseConf, slowThresholdMillisecond int64) (*gorm.DB, error) {
    db, err := gorm.Open(mysql.Open(dsn), gormCfg)
    if err != nil {
        return nil, err
    }
    
    sqlDB, err := db.DB()
    if err != nil {
        return nil, err

    // 优化连接池配置

    sqlDB.SetConnMaxLifetime(time.Duration(conf.MaxIdleTime) * time.Second)  // 连接最大生命周期

}

### 5.2 Redis优化

**Pipeline批量操作**：
```go
func (r *preSecKillStockRepo) BatchGetSecKillInfo(ctx context.Context, secNums []string) ([]*biz.PreSecKillRecord, error) {
    pipe := r.rdb.Pipeline()
    cmds := make([]*redis.StringCmd, len(secNums))
    
    for i, secNum := range secNums {
        cmds[i] = pipe.Get(ctx, secNum)
    }
    
    _, err := pipe.Exec(ctx)
    if err != nil && err != redis.Nil {
        return nil, err
    }
    
    records := make([]*biz.PreSecKillRecord, 0, len(secNums))
    for _, cmd := range cmds {
        val, err := cmd.Result()
        if err == nil {
            var record biz.PreSecKillRecord
            if err := json.Unmarshal([]byte(val), &record); err == nil {
                records = append(records, &record)
            }
        }
    }
    
    return records, nil
}
```
